# 연결 후 파라미터 자동 읽기 — 설계 및 변경 이력

> 최종 업데이트: 2026-04-02

---

## 1. 개요

IMS-DS-16 장비에 시리얼 포트 연결이 완료되면, 장비에 현재 설정된 파라미터를 자동으로 읽어와 프로그램 UI에 표시하는 기능.

---

## 2. 상태 머신 (ReadState enum)

```cpp
enum ReadState
{
    READ_IDLE = 0,           // 대기
    READ_DEVICE_DATA,        // :00R 응답 대기 (ON TIME + MaxPage + 반복 + Section)
    READ_MULTIPLIER,         // :1RU 응답 대기 (배수)
    READ_CUR_PAGE,           // :00G 응답 대기 (현재 페이지)
    READ_PAGE_ONTIME,        // :00R 응답에서 특정 페이지만 읽기
};
```

---

## 3. 연결 시 자동 읽기 시퀀스

```
연결 성공
    │
    ▼
ReadDeviceParams()
    │
    ├─ state = READ_DEVICE_DATA
    ├─ Send(":00R\r\n")  ──►  장비
    │
    ▼
:00R 응답 수신 (여러 줄, CRLF 있음)
    ├─ MAX PAGE : N         → Max Page UI 반영
    ├─ PAGE : NN = ...,RN   → ON TIME + 반복 횟수 UI 반영
    ├─ Section Control:ON   → Section 상태 UI 반영
    ├─ Page:N~N             → Section 범위 UI 반영
    │
    ▼
타이머(100) 300ms 만료 → :00R 수신 완료 판정
    │
    ├─ state = READ_MULTIPLIER
    ├─ Send(":1RU\r\n")  ──►  장비
    │
    ▼
:1RU 응답 수신 (CRLF 없음 — 타이머(101) 500ms로 버퍼 강제 파싱)
    ├─ ACK + 콤마 구분 배수값
    ├─ 16채널 배수 UI 반영
    │
    ▼
state = READ_IDLE — 완료
```

---

## 4. 수동 조회 버튼

| 버튼 | 명령어 | 상태 | CRLF | 타이머 |
|------|--------|------|------|--------|
| [읽기] | `:00R` | READ_PAGE_ONTIME | 있음 | 100 (300ms) |
| [배수 읽기] | `:1RU` | READ_MULTIPLIER | **없음** | 101 (500ms) |
| [CP] | `:00G` | READ_CUR_PAGE | **없음** | 102 (500ms) |
| [MP] | `:1P[MP]` | (전송만) | - | - |

---

## 5. CRLF 없는 응답 처리

일부 명령의 응답이 CRLF 종단 없이 ACK 직후 데이터가 바로 옴.
수신 버퍼에 데이터가 쌓이지만 LF가 없어 ProcessReceivedLine이 호출되지 않는 문제.

**해결:** 타이머 기반 버퍼 강제 파싱

| 타이머 ID | 대상 상태 | 대기 시간 | 동작 |
|-----------|----------|-----------|------|
| 100 | READ_DEVICE_DATA | 300ms | :00R 수신 완료 감지 → 배수 읽기 전환 |
| 101 | READ_MULTIPLIER | 500ms | 버퍼 강제 파싱 or "응답 없음" 건너뜀 |
| 102 | READ_CUR_PAGE | 500ms | 버퍼 강제 파싱 or "응답 없음" 건너뜀 |

---

## 6. :00R 응답 파싱 (Parse00RPageLine)

```
PAGE : 00 = 000,000,...,000,R1
       ^^   ^^^^^^^^^^^^^^^^^^^
       페이지  16CH ON TIME   반복
```

- `PAGE` 접두어 확인 → `=` 기준으로 분리
- 끝에서 `,RN` 추출 → 반복 횟수
- 나머지 콤마 구분 → 16채널 ON TIME 값

---

## 7. RAW 데이터 로그

OnSerialReceive에서 수신 바이트를 HEX + ASCII로 동시 표시.

```
[RAW] 06 38 54 72 | .8Tr
```

장비 응답 형식 분석용. 프로토콜 디버깅 완료 후 제거 가능.

## 9. 변경 이력

### 2026-04-02 (v3) — :00R 전환 + 응답 형식 대응

| 변경 | 내용 |
|------|------|
| 초기 읽기 `:1RA` → `:00R` 전환 | 한 번에 MaxPage + ON TIME + 반복 + Section 읽기 |
| Parse00RPageLine 추가 | `PAGE : NN = ...,RN` 형식 파싱 |
| ParsePageLine 유지 | `PageN:...` 형식 (`:1RA` 호환) |
| ReadState enum 도입 | 매직넘버 0~4 → READ_IDLE, READ_DEVICE_DATA 등 |
| CRLF 없는 응답 처리 | `:1RU`, `:00G` — 타이머 기반 버퍼 강제 파싱 |
| RAW 로그 추가 | HEX + ASCII 동시 표시로 프로토콜 디버깅 |
| [CP] 버튼 추가 | `:00G` 현재 페이지 수동 조회 |
| [MP] 버튼 추가 | `:1P[MP]` Max Page 단독 설정 |
| [배수 읽기] 버튼 추가 | `:1RU` 배수 수동 조회 |
| 장비 설정 상태 영역 개편 | 반복 표시 추가, 조회 가능 항목 녹색으로 구분 |
| Win32 구성 제거 | .sln, .vcxproj에서 x64만 유지 |

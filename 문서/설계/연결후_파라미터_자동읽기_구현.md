# 연결 후 파라미터 자동 읽기 — 계획 및 진행 사항

## 1. 개요

IMS-DS-16 장비에 시리얼 포트 연결이 완료되면, 장비에 현재 설정된 **16채널 ON TIME 값**과 **16채널 배수 값**을 자동으로 읽어와 프로그램 UI에 표시하는 기능.

---

## 2. 변경 전 문제점

| 문제 | 설명 |
|------|------|
| 응답 파싱 없음 | OnSerialReceive()가 ACK/NAK만 인식, 데이터 응답을 파싱하지 못함 |
| 수신 버퍼 없음 | 응답이 분할 수신될 수 있는데 누적 버퍼가 없어 불완전한 데이터 처리 |
| 자동 읽기 없음 | 연결 후 수동으로 [데이터 확인] 버튼을 눌러야만 조회 가능 |
| 상태 관리 없음 | 어떤 명령에 대한 응답인지 추적 불가 |

---

## 3. 구현 계획

### 3.1 동작 흐름

```
연결 성공 (수동 연결 또는 자동 찾기)
    │
    ▼
ReadDeviceParams()
    │
    ├─ m_nReadState = 1 (WAITING_ONTIME)
    ├─ Send(":1RA\r\n")  ── ON TIME 조회 명령 ──►  장비
    │
    ▼
OnSerialReceive() ◄── 장비 응답 수신
    │
    ├─ 수신 바이트를 m_strRecvBuffer에 누적
    ├─ LF(0x0A) 감지 시 한 줄 완성
    │
    ▼
ProcessReceivedLine()
    │
    ├─ m_nReadState == 1 → ParseOntimeResponse()
    │   ├─ "255,000,100,..." → int arrValue[16] 파싱
    │   ├─ SetDlgItemInt()로 CH01~CH16 에디트박스 반영
    │   │
    │   ├─ m_nReadState = 2 (WAITING_MULTIPLIER)
    │   └─ Send(":1RU\r\n")  ── 배수 조회 명령 ──►  장비
    │
    ├─ m_nReadState == 2 → ParseMultiplierResponse()
    │   ├─ "5,1,1,..." → int arrMul[16] 파싱
    │   ├─ CComboBox::SetCurSel()로 배수 콤보박스 반영
    │   └─ m_nReadState = 0 (IDLE) — 완료
    │
    └─ m_nReadState == 0 → 일반 로그 표시만
```

### 3.2 상태 머신

| 상태 | 값 | 의미 | 전이 조건 |
|------|---|------|----------|
| IDLE | 0 | 대기 상태 | 초기값, 파라미터 읽기 완료 시 |
| WAITING_ONTIME | 1 | ON TIME 응답 대기 | ReadDeviceParams() 호출 시 |
| WAITING_MULTIPLIER | 2 | 배수 응답 대기 | ON TIME 파싱 성공 시 |

## 4. 수정 파일 및 변경 내용

### 4.1 ProtocolBuilder.h

```diff
+ static BOOL ParseOntimeResponse(const CString& strLine, int arrValue[16]);
+ static BOOL ParseMultiplierResponse(const CString& strLine, int arrMul[16]);
```

### 4.2 ProtocolBuilder.cpp

- `ParseCommaSeparatedValues()` — 콤마 구분 문자열에서 최대 16개 정수를 추출하는 헬퍼 함수
- `ParseOntimeResponse()` — 응답 라인에서 첫 숫자 위치를 찾아 16채널 ON TIME 값 파싱
- `ParseMultiplierResponse()` — 응답 라인에서 16채널 배수 값 파싱

### 4.3 LightControllerDlg.h

```diff
+ CString m_strRecvBuffer;    // 수신 바이트 누적 버퍼
+ int m_nReadState;            // 0=IDLE, 1=WAITING_ONTIME, 2=WAITING_MULTIPLIER
+ void ReadDeviceParams();     // 읽기 명령 순차 전송
+ void ProcessReceivedLine(const CString& strLine);  // 한 줄 응답 파싱 + UI 반영
```

### 4.4 LightControllerDlg.cpp

| 함수 | 변경 내용 |
|------|----------|
| 생성자 | `m_nReadState(0)` 초기화 추가 |
| `OnBnClickedConnect()` | 연결 성공 후 `ReadDeviceParams()` 호출 추가 |
| `OnAutoFindDone()` | 자동 연결 성공 후 `ReadDeviceParams()` 호출 추가 |
| `OnSerialReceive()` | **전면 재작성** — 수신 버퍼 누적 방식으로 변경, LF 감지 시 `ProcessReceivedLine()` 호출 |
| `ReadDeviceParams()` | **신규** — 상태를 WAITING_ONTIME으로 설정 후 `:1RA\r\n` 전송 |
| `ProcessReceivedLine()` | **신규** — 상태에 따라 파싱 함수 호출, UI 반영, 다음 명령 전송 |

## 5. 사용되는 프로토콜

| 순서 | 명령 | 프로토콜 | 용도 |
|------|------|---------|------|
| 1 | ON TIME 조회 | `:1RA\r\n` | 현재 페이지의 16채널 ON TIME 값 조회 |
| 2 | 배수 조회 | `:1RU\r\n` | 16채널 배수(1~5) 값 조회 |

응답 형식: 콤마로 구분된 숫자 (예: `255,000,100,050,...`)

### Phase 3: 장비값 읽기전용 표시 영역 추가

기존 입력 UI와 별도로, 장비에서 읽어온 값을 읽기전용으로 표시하는 영역을 오른쪽에 추가한다.

```
┌─ 페이지/채널 설정 ──────────────┐ ┌─ 현재 장비값 (읽기전용) ─────┐
│ CH  ON TIME  배수 │ CH  ON TIME │ │ CH  ON TIME  배수             │
│ 01 [500  ] [3▼]  │ 09 [000  ]  │ │ 01  500      x3               │
│ 02 [300  ] [1▼]  │ 10 [000  ]  │ │ 02  300      x1               │
│ ...               │ ...         │ │ ...                           │
│  ← 편집 가능 →    │             │ │  ← 읽기전용, 장비 실제값 →   │
└───────────────────┴─────────────┘ └──────────────────────────────┘
```

| # | 단계 | 상태 | 비고 |
|---|------|------|------|
| 1 | resource.h에 IDC_STC_DEV_ONTIME/MUL ID 추가 | ✅ 완료 | 1500~1515, 1520~1535 |
| 2 | LightController.rc에 읽기전용 그룹 + 32개 Static 추가 | ✅ 완료 | 다이얼로그 너비 540→720 |
| 3 | ProcessReceivedLine에서 장비값 영역 업데이트 | ✅ 완료 | SetDlgItemText로 반영 |
| 4 | x64 빌드 및 확인 | ✅ 완료 | 빌드 성공 |

| # | 항목 | 방법 | 예상 결과 |

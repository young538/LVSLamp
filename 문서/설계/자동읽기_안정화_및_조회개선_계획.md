# 자동 읽기 안정화 및 조회 개선 계획

## 1. 원칙

| 구분 | 목표 | 내용 |
|------|------|------|
| **자동 읽기** | 연결 직후 일관된 조회 | 연결 성공 후 ON TIME, 배수, 현재 페이지를 순차적으로 읽어 상태바와 장비값 영역에 반영 |
| **수동 조회** | 자동 읽기와 동일한 처리 | 데이터 확인, 배수 확인, 현재 페이지 확인, 특정 페이지 읽기가 동일한 상태 머신과 파싱 흐름 사용 |
| **응답 검증** | 불완전한 데이터 차단 | 16채널 응답이 완전하게 수신되기 전에는 UI를 갱신하지 않음 |
| **실패 복구** | 상태 꼬임 방지 | NAK, 무응답, 연결 해제, 잘못된 형식 응답 시 읽기 상태와 UI를 즉시 정리 |
| **표시 구분** | 사용자 혼동 방지 | 편집값, 장비 실제값, 마지막 전송값을 서로 다른 역할로 유지 |

## 2. 안정화 / 개선 분류

### 즉시 수정 필요

| 항목 | 현재 문제 | 개선 방향 |
|------|----------|----------|
| 수동 조회 버튼 | 상태 머신과 분리되어 로그만 남고 UI 반영이 누락될 수 있음 | 자동 읽기와 같은 상태 전이로 통합 |
| 응답 파싱 성공 기준 | 값 1개만 있어도 성공 처리되어 잘린 응답이 반영될 수 있음 | 16채널 완전 수신 기준으로 강화 |
| 읽기 중 명령 처리 | 다른 명령이 끼어들어 상태가 꼬일 수 있음 | 읽기 중 버튼 잠금 또는 명령 거부 정책 도입 |
| 실패 후 정리 | NAK나 해제 후 장비 표시 영역이 이전 값을 유지할 수 있음 | 내부 상태와 장비 표시 영역 동시 초기화 |

### 구조 개선

| 항목 | 현재 상태 | 개선 방향 |
|------|----------|----------|
| 읽기 상태 표현 | 정수 상수와 주석 기반 관리 | enum 수준으로 의미를 명확히 정의 |
| 상태바 표시 | 편집 페이지와 장비 실제 페이지 의미가 섞일 수 있음 | 상태바 표시 기준을 장비 실제 상태 중심으로 정리 |
| 조회 흐름 | 자동 읽기와 페이지별 읽기 로직이 분산됨 | 단일 상태 머신 흐름으로 정리 |
| UI 정책 | 장비값과 마지막 전송값 역할은 분리되어 있으나 갱신 기준이 흩어져 있음 | 갱신 시점과 책임 위치를 명확히 분리 |

### 장비 검증 필요

| 항목 | 현재 가정 | 검증 필요 내용 |
|------|----------|----------------|
| `:00R` 응답 형식 | 페이지 0부터 순차 라인이 온다고 가정 | 실제 장비가 헤더, 종료 라인, 페이지 표기를 포함하는지 확인 |
| 자동 찾기 판정 | ACK 또는 숫자만 보여도 장비로 간주 | 다른 시리얼 장치와 오탐 여부 확인 |
| 현재 페이지 응답 | 숫자 추출만으로 페이지 판정 | 실제 응답 형식이 항상 숫자 중심인지 확인 |

## 3. 목표 동작 흐름

```
연결 성공
    │
    ▼
ReadDeviceParams()
    │
    ├─ WAITING_ONTIME
    ├─ :1RA 전송
    │
    ▼
OnSerialReceive()
    │
    ├─ 수신 버퍼 누적
    ├─ LF 감지 시 한 줄 완성
    │
    ▼
ProcessReceivedLine()
    │
    ├─ WAITING_ONTIME
    │   ├─ 16채널 ON TIME 완전 수신 확인
    │   ├─ 입력 UI + 장비값 영역 반영
    │   └─ WAITING_MULTIPLIER → :1RU 전송
    │
    ├─ WAITING_MULTIPLIER
    │   ├─ 16채널 배수 완전 수신 확인
    │   ├─ 입력 UI + 장비값 영역 반영
    │   └─ WAITING_PAGE → :00G 전송
    │
    ├─ WAITING_PAGE
    │   ├─ 현재 페이지 파싱
    │   ├─ 상태바 + 장비 페이지 표시 반영
    │   └─ IDLE 복귀
    │
    └─ 오류/NAK/타임아웃
        ├─ 상태 초기화
        ├─ 장비값 표시 정리
        └─ 상태바 오류 또는 대기 상태 반영
```
